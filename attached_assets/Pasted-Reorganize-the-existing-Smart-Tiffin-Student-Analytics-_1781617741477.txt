Reorganize the existing Smart Tiffin & Student Analytics Platform without changing any functionality.

This is a full-stack SaaS application that supports two independent user roles:

- Student
- Tiffin Service Owner

The application includes authentication, dashboards, expense tracking, budget management, meal tracking, customer management, billing, payment tracking, analytics, and reporting features.

REORGANIZATION REQUIREMENTS

Move all React, TypeScript, Vite, TailwindCSS, ShadCN UI, dashboard, chart, and frontend-related code into:

frontend/src

Move all Express, Node.js, Prisma, API, authentication, middleware, controller, and backend-related code into:

backend/src

Maintain a simple architecture:

Route → Controller → Service → Database

Do NOT introduce complex architecture patterns such as:

- Domain Layer
- Repository Pattern
- Infrastructure Layer
- Use Cases
- Adapters
- CQRS

SAFE MIGRATION PROCESS

Before making any changes:

Generate a detailed migration plan containing:

1. Current project structure analysis
2. Proposed project structure
3. Files and folders to move
4. Import path changes
5. Configuration files requiring updates
6. Potential risks
7. Rollback strategy
8. Verification checklist

Do NOT delete, overwrite, or modify existing code until the new structure has been verified working.

Maintain a backup-safe migration process.
